# AI 提示词规范与功能说明

本文档总结 SoloAvalon 当前 AI 提示词的设计边界、配置能力和维护约定。目标是让模型像合法玩家一样参与讨论和决策，同时避免把调试信息、配置元信息或隐藏真相泄露给模型。

## 核心原则

- 只使用当前玩家合法可见的信息，包括自己的身份视角、公开玩家列表、公开发言、公开投票结果、任务结果和当前可执行动作。
- 不向模型发送原始 `private_view`、`public_state`、`recent_public_events`、`legal_actions` 等 JSON 上下文字段。
- 不向模型发送提示词版本号、配置键名、默认开关、推荐身份组合等元信息。
- 前缀内容尽量稳定，便于模型服务端缓存；动态信息集中在玩家视角、公开记录和本次行动中。
- 后端是唯一规则裁判。模型只能提出动作意图，合法性由后端校验。

## 配置文件

可提交的提示词配置位于：

```text
config.example/prompt_templates.json
```

本地可通过环境变量覆盖：

```powershell
$env:SOLOAVALON_PROMPT_CONFIG="E:\path\to\prompt_templates.json"
```

真实模型配置仍位于被 git 忽略的 `config/llm_profiles.json`，不要把真实 API Key 或供应商配置写入提示词配置。

## 发送给模型的结构

当前模型消息分为四段：

```text
【系统】
【本局配置】

【你的视角】

【公开记录】

【本次行动】
```

`【系统】` 和 `【本局配置】` 放在 system message 中，用于提供稳定规则背景。`【你的视角】`、`【公开记录】`、`【本次行动】` 分别作为 user message 发送。

### 系统与本局配置

包含：

- 玩家人数。
- 好人/恶方人数。
- 每轮任务人数。
- 每轮任务失败所需失败票数。
- 本局实际身份和身份机制说明。
- 已启用的扩展机制，例如湖中仙女。

不包含：

- `Prompt 模板版本`。
- `推荐身份组合`。
- 配置开关名称，例如 `enabled`。
- 默认启用人数、默认不加入推荐组合等配置管理说明。
- 未启用的可选身份或机制。

### 你的视角

包含：

- 当前玩家 id。
- 当前玩家身份。
- 当前玩家阵营。
- 当前身份玩法。
- 当前身份策略。
- 当前身份合法可见的额外信息。
- 公开玩家 id 列表。

不同身份的额外信息不同：

- 梅林：可见非莫德雷德恶方玩家。
- 派西维尔：可见梅林候选，通常包含梅林和莫甘娜。
- 恶方：可见恶方同伴；奥伯伦不参与恶方互认。
- 崔斯坦/伊索尔德：可互相确认对方为好人。
- 普通忠臣：无额外隐藏信息。

### 公开记录

公开记录按时间顺序整理，只保留正常玩家能看到的信息：

- 组队：队长和车队成员。
- 发言：玩家 id 和公开发言文本。
- 投票结果：车队是否通过，具体谁赞成、谁反对。
- 任务行动提交：只显示哪些任务成员已提交，不显示具体行动。
- 任务结果：成功/失败，以及成功票、失败票数量。
- 刺杀：刺客目标和结算后的胜者。

未结算投票不会公开票型；结算后才展示具体赞成/反对玩家。

### 本次行动

`【本次行动】` 用配置中的 `action_prompts` 约束模型输出 JSON。不同阶段的契约不同：

- 组队：返回车队、公开组队理由和私下判断摘要。
- 发言：返回公开发言和私下判断摘要。
- 投票：返回 `approve` 或 `reject`，不需要公开理由。
- 任务：恶方可选择成功/失败；好人任务行动由后端直接提交成功，不请求模型。
- 刺杀：返回刺杀目标、候选排序和私下判断摘要。

发言阶段的 `stance` 不是必需字段；解析层会在缺省时按 `uncertain` 处理。

## 角色与身份提示

角色基础文案分为三层：

- `role_descriptions`：本局配置中的身份机制说明。
- `role_gameplay`：玩家视角中的身份玩法。
- `role_strategy_tips`：玩家视角中的简短身份策略。

这些文案会按当前玩家身份展示，不会把其他玩家的隐藏身份真相暴露给当前玩家。

当前覆盖身份包括：

- 亚瑟的忠臣 / 忠臣。
- 梅林。
- 派西维尔。
- 刺客。
- 莫甘娜。
- 莫德雷德。
- 奥伯伦。
- 爪牙。
- 崔斯坦。
- 伊索尔德。

## 详细身份提示开关

配置项：

```json
"role_tip_detail": {
  "enabled": false,
  "label": "详细身份提示",
  "role_tips": {}
}
```

默认关闭。关闭时，模型只收到简短身份玩法和身份策略。

打开后，只会给当前玩家自己的身份追加详细 tip，例如更细的发言方式、隐藏方式、保护方式、刺杀判断方式等。模型不会看到 `role_tip_detail`、`enabled` 等配置键名。

详细 tip 适合用于：

- 让新角色扮演更稳定。
- 增强不同身份的发言差异。
- 让模型更接近玩家攻略中的常见打法。

详细 tip 不适合写入：

- 配置说明。
- “默认开启/关闭”之类元信息。
- 当前玩家不应知道的隐藏身份真相。
- 与当前身份无关的其它身份私密策略。

## 可选机制

配置项 `optional_mechanics` 用于描述可开关机制。

当前包含：

- 湖中仙女：任务结算后查看一名未被查看过玩家的阵营，并转交湖女。
- 崔斯坦：和伊索尔德互相确认对方是好人。
- 伊索尔德：和崔斯坦互相确认对方是好人。

只有启用后的机制才进入 `【本局配置】`。未启用机制不会显示给模型，避免模型把未生效规则当成当前对局事实。

## 推荐身份组合

`recommended_role_setups` 用于配置 5-10 人推荐身份组合，供建局和后续开关逻辑使用。

该字段不直接发送给模型。模型只需要知道“本局实际身份”，不需要知道推荐表、默认表或配置来源。

当前推荐组合不包含崔斯坦/伊索尔德。

## 元信息禁止进入模型输入

以下内容不应出现在最终发给模型的 prompt 中：

- 提示词版本号。
- 配置文件路径。
- 配置键名。
- `enabled`、`default_enabled_for_player_counts` 等开关字段。
- `recommended_role_setups` 或“推荐身份组合”。
- “默认不加入推荐身份组合”。
- “可选扩展身份”等配置性质描述。
- 原始 JSON 上下文字段名。

如果需要记录这些信息，应保存在 AI 决策日志或配置文件中，而不是发送给模型。

## 相关代码

- `backend/app/prompting/config.py`：加载和校验提示词配置。
- `backend/app/prompting/templates.py`：生成最终模型 messages。
- `backend/app/ai/context.py`：构造合法玩家视角和稳定前缀。
- `backend/app/prompting/schemas.py`：解析和校验模型 JSON 输出。
- `config.example/prompt_templates.json`：可提交的默认提示词配置。

## 维护建议

- 修改提示词展示结构时，优先检查最终生成的 `prompt_messages`，不要只看配置文件。
- 增加角色时，至少补齐身份标签、机制说明、玩法、策略、合法视角规则。
- 增加开关时，区分“配置给系统看的内容”和“模型需要知道的游戏事实”。
- 测试应优先覆盖长期行为，例如合法视角、不泄露原始 JSON、投票和任务结果可读，而不是绑定某一句攻略文案。
