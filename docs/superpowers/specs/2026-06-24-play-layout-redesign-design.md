# 游玩界面布局重设计

## 背景

当前“对局”页在准备和游玩期间共用同一个三栏布局：左侧开局表单、中间游玩内容或房间列表、右侧行动面板。进入房间后，开局表单仍常驻左侧，导致真实游玩区域被压缩；同时，本轮总结没有独立滚动，信息流窗口的可用垂直区域偏少。

本设计聚焦前端游玩界面布局，不改变后端规则裁判、对局数据结构、AI 决策、房间生命周期或事件可见性。

## 目标

- 游玩期间隐藏创建房间/开局表单，让用户专注当前房间。
- 准备态保留现有开局表单和可继续房间列表，避免影响新建房间流程。
- 扩大信息流的水平和垂直可用区域。
- 让本轮总结拥有独立滚动区域，不再把行动栏或页面整体撑长。
- 保留“返回房间”语义：退出当前房间视图，但不停止、删除或归档房间。

## 非目标

- 不重做房间管理页。
- 不改变信息流内容、事件排序、逐条显现节奏或底部跟随逻辑。
- 不改变行动按钮、提交流程、AI 代打和错误重试行为。
- 不新增后端 API。

## 设计选择

采用“游玩专注模式”。

无 active game 时，对局页继续显示准备态：左侧开局表单，中间可继续房间列表。创建或进入房间后，对局页切换为专注游玩态：开局表单不渲染，主区域只保留当前对局桌面、信息流、行动和总结。

曾考虑但不采用的方案：

- 折叠开局表单：能减少干扰，但仍占用宽度，不能最大化信息流区域。
- 左侧改为房间导航：有利于多房间切换，但当前主要痛点是游玩空间不足，收益不如专注模式直接。

## 界面结构

### 准备态

触发条件：`tab === "game"` 且没有 `game`，并且不在 `startingGame`。

结构保持现状：

- 左侧：开局表单，包括默认模型、玩家人数、昵称、AI 名称和选项。
- 中间：未归档房间列表，可继续游玩或归档。
- 右侧：无行动栏。

### 创建中

触发条件：`startingGame === true`。

创建中使用专注布局，不显示开局表单。主栏显示 `StartingGameDesk`，用于反馈对局正在创建；右栏显示轻量状态面板，保持专注布局宽度稳定，避免创建期间页面先出现开局表单再跳走。

### 游玩态

触发条件：`game !== null`。

新增专注布局容器 `play-focus-layout`：

- 主栏：`GameDesk`，内部继续承载玩家座位、任务轨道和 `GameFlow`。
- 右栏：`action-panel`，包括状态条、当前可执行行动、本轮总结。

桌面端建议使用两栏：

```text
┌───────────────────────────────┬──────────────┐
│ 对局桌面 + 信息流              │ 行动 + 总结   │
└───────────────────────────────┴──────────────┘
```

移动端保持单列：

```text
对局桌面
行动
信息流/总结随组件自然排列
```

## 滚动策略

### 信息流

`information-feed-list` 保留内部滚动，并从固定最大高度调整为基于视口的高度约束。专注布局中，信息流应明显高于当前三栏模式；桌面端使用 `max-height: clamp(520px, calc(100vh - 360px), 760px)` 作为起点，兼顾常见笔记本屏幕和大屏空间。

现有自动滚动行为继续保留：

- 用户已在底部时，新信息出现后跟随到底部。
- 用户向上查看历史时，新信息不强制打断阅读位置。

### 本轮总结

`action-summary-panel` 设置 `max-height` 和 `overflow: auto`。总结卡片多时只滚动总结区域，不挤压上方状态条和行动按钮，也不迫使整个页面不断变高。

## 组件边界

- `App` 负责根据 `game` 和 `startingGame` 选择准备态或专注游玩态。
- `PlayableRoomList` 继续只用于准备态。
- `GameDesk` 保持当前职责：展示对局桌面和信息流，不承担开局表单逻辑。
- `ActionRoundSummary` 只调整容器滚动样式，不改变总结数据来源。

如实现时发现 JSX 过长，可抽出小的渲染函数或组件来表达准备态和游玩态，但不做无关重构。

## 响应式行为

- 大屏：准备态可继续使用现有三栏；游玩态使用两栏，右侧行动栏保持固定或窄范围宽度，主栏占据剩余空间。
- 中等屏：游玩态可降为单列或主栏优先、行动栏置后，避免两栏过窄。
- 小屏：所有内容单列，按钮和信息卡片不得溢出容器。

## 测试与验证

前端源码断言测试需要覆盖：

- 游玩态存在专注布局标识 `play-focus-layout`。
- 开局表单只在准备态渲染，游玩态不与 `GameDesk` 同时出现在同一个三栏结构中。
- 总结栏样式包含独立滚动约束。
- 信息流样式使用更大的视口相关高度约束，并继续保留内部滚动。

```powershell
cd frontend

```
